Special Commands

 

Special commands can be enabled in the setup portal. These are advanced settings that is intended for advanced users. They are typed into the portal field labeled special commands. DeviceAway also has a full command line interface that can set special commands and access all settings. Use a dumb terminal and connect to the Wall USB port at 115,200 baud. Type the word help to see a list of all the CLI commands. Any setup changes in the CLI require the command saveall to store in flash.

 

beta - Enables access to all beta versions of the firmware. Beta testers play a crucial role in identifying and resolving bugs before the official release. Anyone interested in becoming a beta tester can do so by entering a special command in the commands field of the setup portal.

 

cc:support@DeviceAway.net - used for beta testing and sending a copy of all emails to the factory.

 

debug - case insensitive. Adds debug information to every email.

 

graph10ms – This command instructs the system to send four emails per hour to the factory. Each email contains 10ms current ADC sample data in a graph format, providing detailed analysis of the charger’s performance.   Graphing stops after 24 hours.

graph1sec – This command instructs the system to send a single email every 24 hours to the factory.   The email contains 1 second current ADC sample data in a graph format.   Graphing will continue for any units with the word beta in the special commands.

server: yourservername port: yourserverport sender: loginname password: yourpassword - sets the login credentials for outgoing emails. By default, DeviceAway uses a variety of smtp email servers to send notification emails. This allows redundancy when one server can not send the email, multiple backups will. Entering in a custom email server login allows the first attempt at sending an email to always use a server that you provide. SSL is enabled always.
